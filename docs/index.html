NemoClaw DGX cmd_vel Bridge
Install Modes Voice GitHub
ROS 2 /cmd_vel control through DGX Spark

Voice control for Wheeltec and Jetson robot cars.

A small REST bridge lets NemoClaw send safe timed motion commands to one or more physical robot cars over ROS 2. Use normal LAN discovery when the cars are nearby, or Tailscale with FastDDS discovery when they are remote.

Start deploying View repository
NemoClaw / sandbox Voice or TUI command
↓ REST /command
DGX Spark Flask bridge publishes ROS 2 Twist
↓ /cmd_vel
Wheeltec / Jetson car(s) Subscribe and drive motors

Quick Install

Install the DGX or Jetson launch scripts directly from GitHub.

DGX Spark 
curl -fsSL https://raw.githubusercontent.com/rennn0223/nemoclaw-dgx-cmdvel-bridge/main/install.sh | \
  ROLE=dgx GITHUB_REPO=rennn0223/nemoclaw-dgx-cmdvel-bridge bash
Jetson / Wheeltec 
curl -fsSL https://raw.githubusercontent.com/rennn0223/nemoclaw-dgx-cmdvel-bridge/main/install.sh | \
  ROLE=jetson GITHUB_REPO=rennn0223/nemoclaw-dgx-cmdvel-bridge bash

Choose A Network Mode

Use one mode at a time. Do not mix LAN discovery with the Tailscale-only FastDDS profile.

Mode A: Same Wi-Fi / LAN

Use when DGX can already see car topics with ros2 topic list -t.

# DGX
cd /home/nvidia
./auto_lan.sh

# Jetson
cd /home/wheeltec
./wheeltec_bringup_lan.sh

Mode B: Different Networks / Tailscale

Use when the cars are remote or LAN multicast discovery does not work.

# DGX
cd /home/nvidia
./auto_tailscale.sh

# Jetson
cd /home/wheeltec
./wheeltec_bringup_tailscale.sh

Install The NemoClaw Skill

Install the voice-friendly skill into the sandbox from DGX.

nemoclaw my-assistant skill install /home/nvidia/cmd-vel-jetson-car

Fallback: upload the skill folder to /sandbox/.openclaw/skills/cmd-vel-jetson-car.

iPhone Voice Commands

Test And Diagnose

Start with stop, then use a one-second motion test.

REST checks

curl http://100.64.0.4:5000/health
curl -X POST http://100.64.0.4:5000/stop
curl -X POST http://100.64.0.4:5000/command \
  -H "Content-Type: application/json" \
  -d '{"linear_x":0.1,"angular_z":0.0,"seconds":1}'

ROS graph check

ros2 topic info /cmd_vel -v

Expect one publisher from DGX and one subscriber per car. With two cars, Subscription count should be 2.

Safety note

If the car keeps moving, immediately send POST /stop. The bridge and helper both send stop signals, but physical robots should always be tested in a clear area.

Built for NemoClaw, DGX Spark, ROS 2, and Wheeltec/Jetson robot cars.
