NemoClaw DGX cmd_vel Bridge
安裝 / Install 模式 / Modes 語音 / Voice GitHub
透過 DGX Spark 控制 ROS 2 /cmd_vel

用 NemoClaw 語音控制 Wheeltec / Jetson 小車。

這個專案提供一個 DGX Spark REST bridge，讓 NemoClaw 可以把「前進兩秒」、 「右轉五秒」、「停車」這類語音指令轉成 ROS 2 /cmd_vel。
A DGX Spark REST bridge for turning NemoClaw voice commands into safe, timed ROS 2 /cmd_vel motion commands.

開始部署 / Start 查看原始碼 / Repository
NemoClaw / sandbox 語音或 TUI 指令 / Voice or TUI command
↓ REST /command
DGX Spark Flask bridge 發布 ROS 2 Twist / publishes ROS 2 Twist
↓ /cmd_vel
Wheeltec / Jetson car(s) 訂閱並驅動馬達 / subscribe and drive motors

快速安裝 / Quick Install

直接從 GitHub 安裝 DGX 或 Jetson 端啟動腳本。Install launcher scripts directly from GitHub.

DGX Spark
Command
curl -fsSL https://raw.githubusercontent.com/rennn0223/nemoclaw-dgx-cmdvel-bridge/main/install.sh | \
  ROLE=dgx GITHUB_REPO=rennn0223/nemoclaw-dgx-cmdvel-bridge bash
Jetson / Wheeltec
Command
curl -fsSL https://raw.githubusercontent.com/rennn0223/nemoclaw-dgx-cmdvel-bridge/main/install.sh | \
  ROLE=jetson GITHUB_REPO=rennn0223/nemoclaw-dgx-cmdvel-bridge bash

選擇網路模式 / Choose A Network Mode

同一時間只使用一種模式。Use one mode at a time.

模式 A：同 Wi-Fi / LAN

DGX 和小車在同一個 Wi-Fi，且 DGX 已經可以用 ros2 topic list -t 看到小車 topics 時使用。
Use normal ROS 2 LAN discovery when DGX can already see car topics.

Command
# DGX
cd /home/nvidia
./auto_lan.sh

# Jetson
cd /home/wheeltec
./wheeltec_bringup_lan.sh

模式 B：不同網域 / Tailscale

DGX 和小車不在同一網路，或 LAN multicast discovery 不穩時使用。
Use Tailscale + FastDDS discovery for remote or cross-network cars.

Command
# DGX
cd /home/nvidia
./auto_tailscale.sh

# Jetson
cd /home/wheeltec
./wheeltec_bringup_tailscale.sh

安裝 NemoClaw Skill / Install The NemoClaw Skill

從 DGX 將語音友善的小車 skill 安裝到 sandbox。Install the voice-friendly robot skill into the sandbox.

Command
nemoclaw my-assistant skill install /home/nvidia/cmd-vel-jetson-car

備用方式：手動上傳 skill 到 /sandbox/.openclaw/skills/cmd-vel-jetson-car.
Fallback: manually upload the skill folder to the sandbox skills directory.

iPhone 語音指令 / Voice Commands

短句最穩。沒有講秒數時，預設移動一秒。Short phrases work best.

前進
前進兩秒
左轉一秒
右轉五秒
慢慢前進
後退
停車
不要動

測試與診斷 / Test And Diagnose

先測停車，再測一秒短移動。Start with stop, then use a one-second motion test.

REST 檢查 / REST checks

Command
curl http://100.64.0.4:5000/health
curl -X POST http://100.64.0.4:5000/stop
curl -X POST http://100.64.0.4:5000/command \
  -H "Content-Type: application/json" \
  -d '{"linear_x":0.1,"angular_z":0.0,"seconds":1}'

ROS graph 檢查 / ROS graph check

Command
ros2 topic info /cmd_vel -v

預期會看到 DGX 的 publisher，以及每台小車各一個 subscriber。
Expect one DGX publisher and one subscriber per car.

安全提醒 / Safety note

如果小車持續移動，立刻送出 POST /stop。Bridge 和 helper 都有停車保護， 但實體車第一次測試仍應放在安全空間。
If the car keeps moving, immediately send POST /stop. Always test physical robots in a clear area.
